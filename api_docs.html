CryptoPulse API Reference

REST JSON 無需 API Key 更新週期 5 分鐘 v2.1.0

平台定位：source-agnostic 多資產市場資料聚合與詮釋平台。把跨市場（crypto / 美股 / 台股）的事實（news + factors + events）分層提供給下游交易系統，但本身不下交易決策。

層級端點回答的問題
事實層/api/news · /api/macro-events · /api/v2/factors發生了什麼？數字是多少？
詮釋層/api/comment/{asset_class} （coming soon）現在是什麼狀態？為什麼？
決策層— 下游 client 自己負責 —我該做什麼？

資料流架構

API 請求本身不打任何上游。所有資料已由背景 Collector 預先抓取並存入 SQLite；API 只讀資料庫，分類/評分亦全在本地計算。
階段執行者頻率說明
① 資料收集Collector（背景常駐）每 5 分鐘從 jin10 / RSS / FRED / DeFiLlama 等上游抓取，寫入 SQLite
② API 查詢/api/news 等端點即時（毫秒級）從 SQLite 讀取已存資料，本地執行分類（category / event_type / direction_en）與相關性評分（relevance_crypto），組裝回傳

上游失效時，Collector 無法更新 SQLite，sources.*.age_seconds 會增加，但 API 本身仍正常回應已存資料。

時間戳語義

Backtest 請用 available_at（系統可讀到的時間），不要用 published_at（上游發佈時間），否則有 lookahead bias。
欄位語義Backtest 安全
published_at上游發佈時間（jin10 精度：秒）❌ lookahead 風險
available_at / saved_at系統首次可讀到的時間✅ no-lookahead 邊界
computed_atFactor / gate 計算時間✅ 用這個對齊 bar
generatedAt主循環產出時間✅ 整體快照新鮮度

V2 端點均附帶 X-Data-Age-Hours response header。資料超過 8 小時未更新回傳 503。

錯誤格式

HTTP情境
404尚無資料（update-data.mjs 從未執行過）
400缺少必要 Query 參數或格式錯誤
503V2 資料超過 8 小時未更新（stale）
500SQLite 讀寫失敗，見 /api/update-log

事實層端點

GET /news ★ 主要

GET /api/news

Source-agnostic news 端點。永遠從 SQLite 讀取已存資料，不直接打上游。分類（category / event_type）、方向（direction_en）、相關性評分（relevance_crypto）均在本地即時計算。上游失效時 sources.*.age_seconds 會增加，但 API 本身仍正常回應。

Query 參數

參數類型預設說明
limitint30最大 200
sinceISO—cursor，published_at >= since，live polling 用
sourcestring—jin10（逗號分隔，未來多源）
categorystring—crypto / macro / regulatory / geopolitical / commodity / general
event_typestring—見下方列舉
directionstring—bullish / bearish / neutral / ambiguous
relevance_minfloat0.3加密相關性門檻 [0,1]
min_confidenceint—1–5，舊 jin10 confidence 欄位
include_low_relevanceboolfalsetrue = 不套 relevance 過濾

event_type 列舉

值觸發情境
fed_speechFed / FOMC 官員發言、聲明
data_releaseCPI / NFP / PPI / GDP 數據公布
etf_flowETF 資金流入 / 流出（含金額）
liquidation_event清算事件（含金額）
regulationSEC / CFTC 監管動態
trade_policy關稅 / 貿易戰政策
central_bankBOJ / ECB / PBOC 決策
geopolitical_shock戰爭 / 制裁 / 能源供應衝擊
general其他

回應結構

{
  "ok": true,
  "count": 8,
  "sources": {
    "jin10": { "latest_saved_at": "2026-04-08T04:30:00Z", "age_seconds": 142 }
  },
  "filters_applied": { "relevance_min": 0.3, "direction": "bearish", ... },
  "items": [
    {
      "id": "3780450",
      "source": "jin10",
      "source_id": "3780450",
      "source_url": "https://flash.jin10.com/detail/3780450",
      "published_at": "2026-04-08T04:28:12Z",
      "available_at": "2026-04-08T04:30:05Z",    // ← backtest 用這個
      "ingest_lag_seconds": 113,
      "content": "聯準會主席鮑威爾警告：通脹仍高，不急於降息。",
      "is_clickbait": false,
      "relevance_crypto": 0.75,
      "category": "macro",
      "entities": ["Fed", "Powell", "CPI"],
      "event_type": "fed_speech",
      "direction_en": "bearish",                  // ← v2.1.0 改良版
      "classifier_version": "v2.1.0",
      // 舊欄位（保留向後相容）
      "direction": "做空",
      "confidence": 4,
      "commentary": "加息預期升溫，風險資產短線承壓。",
      "is_important": 1,
      "saved_at": "2026-04-08T04:30:05Z"
    }
  ]
}
關於 count=0：source=jin10 為廣泛中文財經快訊（含 A 股、商品期貨、宏觀），非 crypto 專用 feed。預設 relevance_min=0.3 僅保留與加密市場直接相關的新聞；當 jin10 近期無 crypto 快訊時，回傳空陣列屬正常行為。

若需取得宏觀/地緣訊號（對 crypto 有間接影響）：
  • ?include_low_relevance=true：關閉相關性過濾，回傳全部
  • ?category=macro 或 ?category=geopolitical：指定分類
  • ?relevance_min=0.1：降低門檻，納入間接相關新聞
direction_en v2.1.0 修正：舊版在「鷹派 Fed 警告通脹，但同段提及降息」情境會誤判為 bullish。新版先偵測明確鷹/鴿框架，再偵測對照連詞（卻/但/however），主句鷹派優先。
輸出：bullish / bearish / neutral / ambiguous（衝突訊號，不強行給方向）

GET /api/news/history

分頁歷史查詢，backtest replay 必要。支援時間窗 from / to 與游標 since。

Query 參數（在 /news 基礎上新增）

參數類型預設說明
fromISO—時間窗起點（published_at >= from）
toISO—時間窗終點（published_at <= to）
offsetint0分頁偏移
limitint100最大 1000

}

GET /v2/factors ★

GET /api/v2/factors

完整 factor vector（25 個），下游 modeling 的核心輸入。每個 factor 含 score [-1,1]、raw value、direction、confidence、computed_at。

Factor 命名空間

NamespaceFactors計入 composite
macro.*vix · dxy · yield_spread_2s10s · yield_10y · yield_2y · yield_3mvix / dxy / spread / 10y ✅
sentiment.*fear_greed · btc_dominance✅
liquidity.*stablecoin_mcap · stablecoin_change_7d · defi_tvl✅
flows.*etf_net_flow_7d✅
derivatives.*liquidation_7d · btc_funding_rate · btc_open_interest · btc_long_short_ratio · btc_taker_cvd · btc_put_call_ratio · btc_iv✅
risk.*geopolitical_bias · regulatory_bias✅

GET /v2/gates

GET /api/v2/gates

Gate conditions（8 個）。含完整 reason、contributing_factors、confidence。平台提供條件，決策是下游的事。

Gate key類型說明
macro.favorablebool宏觀支持風險偏好（yield 曲線 + VIX）
liquidity.adequatebool穩定幣 > $120B 或 DeFi TVL > $60B
event.blackout_windowbool未來 24h 內有 high-importance 宏觀事件
direction.bullish_biasbool加權多空評分 > 0.15
direction.bearish_biasbool加權多空評分 < -0.15
risk.leverage_overextendedbool7D 清算 > $700M
risk.regulatory_levelstringlow / elevated / high
risk.yield_curve_invertedbool2Y-10Y 殖利率曲線倒掛

Deprecated 端點

以下端點均附帶 Deprecation + Sunset: 2026-10-01 response header。

/api/jin10   /api/jin10/history

替代：GET /api/news / GET /api/news/history（加 ?source=jin10）

/api/v2/trade/signal

替代：GET /api/v2/factors + GET /api/v2/gates（下游自行決策）

移除原因：

  • Symbol-blind / asset-class-blind — 無法服務 per-symbol per-strategy 決策
  • 出現 action 動詞（enter_long / reduce / wait）不屬於資料平台職責
  • 與即將推出的 /api/comment 語義重疊

/api/v2/snapshot   /api/v2/staleness

替代：GET /api/v2/factors + GET /api/v2/gates（staleness 已折進 factors 各欄位的 freshness_seconds）
